Web API 第一天

web API介绍

什么是API？

  1. API(Application Programming Interface,应用程序编程接口)，它是程序和开发人员自建基于某些硬件或者软件得以访问的一组例程的能力，不需要关注内部工作细节，不需要了解工作机制(源码);

    1. 比如

      1. 用锤子将钉子钉在墙上(锤子是工具) API就是工具

      2. JS编程sort()排序方法

        浏览器内部组成 谷歌浏览器为例：

        1. 渲染引擎(webkit)
        2. JS解析器(v8)

    2. API(指的是一大堆方法，不同的方法实现不同的功能)

    3. 我们可以简单理解API是封装好的函数

    4. 只有对象才可以挂在属性和方法

  2. 特点：api有输入和输出的特点;

什么是web API？

  1. 浏览器提供的一些关于操作浏览器窗口以及页面中的元素的方法和属性
  2. webAPI我们学习两部分
    • DOM
    • BOM

JS组成

  1. ECMAscript提供一个js的基础语法核心
  2. DOM是文档提供一堆能够操作页面中的元素的API(工具或者方法和属性)
    1. eg：单击一个按钮后做什么事情
  3. BOM是浏览器提供的一堆能够实现控制浏览器的一些行为的API
    1. eg：弹出一些浏览器的弹出框 alert()
    2. 定时器
    3. 地址栏跳转
    4. 历史记录 刷新 关闭窗口

​

DOM简介

  1. DOM document object model

  2. 文档：整个页面文档 document

  3. 模型：他是一个对于具体事物的一种抽象性描述

  4. 对象：我们将整个文档页面抽象成一个大的对象 document对象，既然是对象，就有很多api

  5. 节点：我们可以把文档页面中的所有标签以及注释，还有属性或者文本都可以堪称一个一个的节点

  6. 文档树：我们可以把页面中的所有的节点通过关系绘制出一个大概的结构，这个结构看起来像一棵树，所以我们将页面中的文档结构堪称文档树结构

  7. 元素：标签

  8. 属性

##DOM里面的一些获取元素的方法

  1. document.getElementById() 在整个文档页面中，通过元素的id名获取元素
    1. 注意 这种方式可以获取id，但是在很多浏览器中也可以直接写id名在js中使用(只有id可以)，但是这个方式不推荐，最好统一使用获取的正规方式
    2. 获取到的元素的类型，是
  2. document.getElementsByTagName() 在整个文档下，通过元素的标签名获取元素
    1. 它获取的是一个html元素的集合列表，是一个类数组(伪数组)，它有数组的下标和length长度，但是没有数组的方法
    2. 获取到的元素集合，是HTMLCollection
    3. 这个方法有一个动态性特点：
      1. 当我们在页面元素生成之前去获取标签元素，它是空的，但是里面却存有多个元素，这些元素是程序后面执行完毕之后，动态往里面添加进去的
    4. 可以从具体的某个元素的里面获取元素.getElementsByTagName
  3. document.getElementsByName()
    1. 注意 它在有些浏览器中可以获取id名的元素，如果前面有相同的name名会发生覆盖
  4. document.getElementsByClassName() 在文档下，通过类名获取页面中的元素
    1. 注意 可以从具体的某个元素的里面获取元素.getElementsByClassName()
    2. 有兼容性问题，IE8及一下的浏览器不支持
      1. 手动封装一个className的函数
      2. 使用jQuery里面的获取类的方法
  5. document.querySelector() 它获取的是对应的选择器选择到的元素，如果是多个，则只获取第一个
  6. document.querySelectorAll() 它获取的是对应的选择器选择到的所有的元素
    1. 获取的是一个集合列表，需要通过下标访问具体元素
    2. 注意 querySelector()和querySelectorAll() 在IE7及以下不支持

事件三要素

  1. 我们在做一些效果的时候，需要去观察和分析其中的事件
    1. js这个语言，是以事件驱动为主的编程语言
    2. 因为摸个事件发生，而导致一个结果
    3. 了解几个常用的事件
      1. mouseover 鼠标移上事件
      2. mouseout 鼠标移出事件
      3. click 单击事件
      4. contextmenu 右击事件
      5. mousemove 鼠标移动事件
      6. mousedown 鼠标按下
      7. mouseup 鼠标弹起事件
      8. keydown 键盘按下事件
      9. keyup 键盘弹起事件
      10. 表单事件
      11. 系统事件
        1. 自定义事件……2亲2 2·1
  2. 写效果的时候，要分析事件三要素
    1. 获取事件相关元素
    2. 考虑事件类型
    3. 绑定事件处理函数

补充 事件处理函数，可以是匿名函数，也可以是有名函数
